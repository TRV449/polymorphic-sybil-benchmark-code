# Dataset Card

## Name

Failure-Mode-Aware Benchmark for Grounded Answer Destabilization under Polymorphic Sybil Poisoning

## Purpose

This benchmark measures how retrieval poisoning destabilizes grounded QA.
It does not assume that all attacks end in exact target hijack.
Instead, it explicitly tracks whether the answer layer:

- retains the gold answer,
- shifts to the attacker's poison target,
- drifts to a third incorrect answer, or
- abstains.

## Task Definition

Input:

- a question from HotpotQA or Natural Questions
- a fixed clean corpus snapshot
- an attached poisoning set generated with polymorphic sybil variants

Output:

- a short grounded answer or abstention

## Threat Model

- Retrieval poisoning with polymorphic sybil documents
- Closed-set worst-case purification stress test
- Attack documents are associated with a known poison target
- The goal is not only to maximize targeted hijack, but also to destabilize grounded answering

## Data Sources

- HotpotQA
- Natural Questions
- Wikipedia-derived corpus snapshot
- Poison documents generated by the polymorphic sybil protocol

## What Is Released

- fixed split manifests
- corpus / index snapshot metadata
- poison generation metadata
- official evaluator
- baseline run scripts
- expected benchmark report structure

## Benchmark Tracks

### Clean-Strong

- uses a dataset-aware clean reader
- targets the clean QA ceiling
- should not be conflated with the fairness reference used inside the attacked/defended pipeline

### Attack/Defense Fairness Track

- keeps Base and Attack on the same reader stack
- attributes score changes to poisoning and purification, not to a stronger clean-only reader

### Pseudo-Oracle Control Track

- provides parametric-only and answer-string-matched pseudo-oracle controls
- should not be over-claimed as dataset-provided gold provenance oracle evidence

## Official Metrics

- `ACC`: eval-answer exact match with gold answer
- `ASR`: eval-answer exact match with poison target
- `CACC`: accuracy among non-abstained answers
- `Abstain Rate`
- `Third-Answer Drift Rate`: answered, not gold, not poison target

## Official Answer Protocol

- Store both `raw_answer` and `eval_answer`
- `eval_answer` is obtained through `extract_first_answer`
- `[ERROR: ...]` style answers are normalized to `Unknown`
- existing `eval_answer` strings that already contain `Unknown`-like markers should also be canonicalized to `Unknown`
- Official scoring uses eval answers only
- `poison_target` must be stored in the CSV so ASR can be recomputed from first principles

## Known Limitations

- The benchmark is a worst-case closed-set stress test, not a full open-web realism benchmark
- Attack strength should not be summarized by ASR alone
- Reader behavior and postprocessing strongly affect observed failure modes
- Chunk retrieval and stronger QA readers may change the clean performance ceiling
- current pseudo-oracle controls are answer-string-matched upper-bound probes, not full provenance-supervised oracle evidence

## Recommended Claims

Use:

- benchmark for grounded answer destabilization
- failure-mode-aware benchmark for retrieval poisoning in grounded QA
- polymorphic sybil poisoning stress test
- closed-set purification stress test with answer-level failure analysis

Avoid:

- first benchmark claim
- always-successful target hijack claim
- universal open-web security claim
